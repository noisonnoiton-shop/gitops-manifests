# GitOps Manifests Repository

이 레포지토리는 GitOps 방식으로 Kubernetes 클러스터를 관리하기 위한 매니페스트 파일들을 포함하고 있습니다.

## 개요

이 레포지토리는 선언적 방식으로 Kubernetes 리소스를 관리하며, ArgoCD를 통해 자동으로 클러스터에 배포됩니다. 모든 환경(개발, 스테이징, 프로덕션)에 대한 Kubernetes 매니페스트가 포함되어 있습니다.

## 레포지토리 구조

```
.
├── base/                  # 기본 매니페스트 파일
│   ├── account/          # Account 서비스 매니페스트
│   ├── order/           # Order 서비스 매니페스트
│   └── product/         # Product 서비스 매니페스트
├── overlays/             # 환경별 설정 오버레이
│   ├── dev/             # 개발 환경 설정
│   ├── staging/         # 스테이징 환경 설정
│   └── prod/            # 프로덕션 환경 설정
└── argocd/              # ArgoCD 애플리케이션 정의
```

## 사용 방법

### 사전 요구사항

- Kubernetes 클러스터
- ArgoCD 설치
- kubectl CLI 도구

### 새로운 서비스 추가

1. `base` 디렉토리에 새로운 서비스 디렉토리 생성
2. 기본 매니페스트 파일 작성 (deployment.yaml, service.yaml 등)
3. 각 환경의 `overlays` 디렉토리에 환경별 설정 추가
4. ArgoCD 애플리케이션 정의 추가

### 배포 프로세스

1. 매니페스트 변경사항을 이 레포지토리에 커밋
2. ArgoCD가 자동으로 변경사항을 감지하고 클러스터에 적용
3. 배포 상태는 ArgoCD UI에서 모니터링 가능

## 환경별 설정

### 개발 환경 (dev)
- 리소스 제한 낮음
- 디버그 모드 활성화
- 자동 배포 설정

### 스테이징 환경 (staging)
- 프로덕션과 유사한 설정
- 테스트 데이터 사용
- 성능 테스트 환경

### 프로덕션 환경 (prod)
- 높은 가용성 설정
- 리소스 제한 엄격
- 수동 승인 프로세스

## 모니터링 및 로깅

- Prometheus를 통한 메트릭 수집
- Grafana 대시보드로 시각화
- EFK 스택을 통한 로그 관리

## 문제 해결

일반적인 문제 해결 방법:

1. ArgoCD UI에서 동기화 상태 확인
2. `kubectl` 명령어로 리소스 상태 확인
3. 애플리케이션 로그 확인

## 기여 가이드

1. 새로운 브랜치 생성
2. 변경사항 작성
3. Pull Request 생성
4. 코드 리뷰 진행
5. 승인 후 main 브랜치에 머지
